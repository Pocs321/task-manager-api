# Debug Skill

When debugging an issue:

1. Reproduce the problem - understand what's expected vs. what's happening
2. Read relevant source files and error messages carefully
3. Trace the data flow from input to the point of failure
4. Identify the root cause, not just the symptom
5. Propose a minimal fix that addresses the root cause
6. Verify the fix doesn't introduce regressions
7. Suggest a test that would catch this bug in the future
